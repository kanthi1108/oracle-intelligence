# Project State: Oracle Intelligence (ATLASIQ)

## Current Architecture Summary

- **Framework**: Next.js 14.2.35 (App Router), TypeScript, Tailwind CSS
- **Database**: Supabase (PostgreSQL) with RLS, service role client for admin ops
- **Auth**: Supabase Auth (email/password + magic link via callback route)
- **Payments**: Razorpay (India) and Stripe (International) webhook handlers
- **AI**: Claude Sonnet (placeholder — no actual API integration)
- **Notifications**: Telegram bot with database fallback logs
- **Build**: Compiles successfully with no errors

## Authentication Status

- Login/Register flow works via Supabase Auth with email/password
- Onboarding route (`/api/auth/onboarding`) creates user profile + grants 150 initial credits
- Callback route (`/callback`) handles OAuth code exchange + Spark tier provisioning
- Middleware protects `/`, `/dashboard/*`, `/api/reports/*`, `/settings` — redirects unauthenticated users
- Email verification enforced in middleware — blocks unconfirmed emails
- Forgot password + reset password flows implemented on UI
- Settings page allows profile updates (name, company) and password change

## Admin Status

- Admin dashboard at `/dashboard` with 4 tabs: Users, Reports, Locations, Settings
- Server-side layout (`(admin)`) gates access by checking role in DB
- Admin can: change user roles, change subscription plans, delete non-admin users, toggle locations, credit override modal
- Admin audit logging to `admin_audit_log` table partially implemented
- Admin settings tab allows adjusting platform weights (saved to `platform_weights` table)

## Report Generation Status

- **Client-side engine** (`useOracleEngine` hook) performs all scoring in-memory:
  - Normalized weighted scoring per PRD §3.2
  - Variance matrix calculation
  - Flip variable detection
  - Causality event feed generation
- **No actual AI narrative generation** — Layer 3 (Strategic Brief) uses hardcoded templates, not Claude API
- **Seed data provides 1 demo report** (Madhapur vs Gachibowli GYM) for judge demo user
- **Report save** (`/api/reports/save`) persists to `reports` table with placeholder AI content
- **Report export** (`/api/reports/export`) generates HTML via template function
- **Report generate** (`/api/reports/generate`) has credit guard but returns placeholder (no actual generation)

## Credit System Status

- Append-only ledger (`credits` table) with `current_credit_balances` view
- Credit consumption on report run via `/api/credits/consume`
- Admin credit allocation via `/api/credits/allocate` with audit logging
- Sandbox purchase (simulated) via `/api/credits/sandbox-purchase` — grants 50 credits
- Credit exhaustion modal blocks further analysis
- Webhook handlers for Stripe and Razorpay credit provisioning (placeholders — no actual payment integration)

## Email Verification Status

- Supabase Auth handles email confirmation
- Middleware blocks unconfirmed emails with redirect to `/login?error=verify_email`
- UI shows "Verify Pending" state after registration
- Forgot password flow uses Supabase `resetPasswordForEmail`
- Reset password page at `/reset-password` updates password via `supabase.auth.updateUser()`

## Password Reset Status

- `/forgot-password` page requests reset link
- `/reset-password` page allows setting new password
- Links properly wired between login page and forgot password
- Uses Supabase built-in password recovery flow

## Telegram Status

- `lib/telegram.ts` has full lifecycle notification system
- Supports: REPORT_GENERATED, CREDIT_LOW, CREDIT_EXHAUSTED, SUBSCRIPTION_ACTIVATED, etc.
- Falls back to database log (`telegram_fallback_logs`) when bot token is the generic placeholder
- Fires on credit allocation, report generation, subscription activation
- Silent failure if user has no telegram_chat_id

## Known Limitations

1. **No actual AI narrative** — Strategic Brief content is hardcoded per business type, not generated by Claude
2. **No actual payment integration** — Razorpay/Stripe keys are placeholders; webhook handlers are written but unreachable
3. **Seed locations are duplicated** — 16 in client-side `SEED_LOCATIONS` array, 40 in DB. Client fetches from DB but falls back to seed
4. **L5 CausalityFeed is a stub** — Original causality event generation logic was removed, component shows "Analysis Methodology" with static lists instead
5. **Demographic Visuals use pseudo-random data** — Gender/age distributions are hash-based fakes, not real data
6. **`/upgrade` page does not exist** — Credit exhaustion modal links to `/upgrade` which would 404
7. **No real-time updates** — Realtime subscription only watches locations table, not reports
8. **Credit consume is NOT idempotent** — Missing `idempotency_key` in `/api/credits/consume` route
9. **sandbox-purchase route uses invalid transaction_type** — `'sandbox_purchase'` is not in the CHECK constraint of the `credits` table
10. **Admin dashboard role/plan changes bypass RLS** — Client-side supabase calls directly, may fail if RLS prevents updates from browser
